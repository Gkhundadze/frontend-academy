💾 Web Storage API

The Web Storage API provides mechanisms by which browsers can store key/value pairs, in a much more intuitive fashion than using cookies.

localStorage vs sessionStorage

Key Difference

localStorage: Persists even when the browser is closed and reopened.
sessionStorage: Is cleared when the page session ends (when the tab is closed).

Basic Storage Usage
// 1. Save data to localStorage
localStorage.setItem('username', 'JohnDoe');

// 2. Read data from localStorage
const name = localStorage.getItem('username');
console.log('Saved name:', name);

// 3. Remove specific item
localStorage.removeItem('username');

// 4. Clear ALL items in localStorage
localStorage.clear();

Storing Objects and Arrays

Web Storage can only store Strings. If you want to store objects or arrays, you must convert them using JSON!

Using JSON.stringify and JSON.parse
const userPrefs = {
  theme: 'dark',
  fontSize: 16,
  notifications: true
};

// ❌ BAD: This will just store the string "[object Object]"
// localStorage.setItem('prefs', userPrefs);

// ✅ GOOD: Convert to string first
localStorage.setItem('prefs', JSON.stringify(userPrefs));

// Read it back and convert it back to an Object
const savedPrefsString = localStorage.getItem('prefs');

if (savedPrefsString) {
  const prefsObj = JSON.parse(savedPrefsString);
  console.log('Restored theme:', prefsObj.theme);
}
