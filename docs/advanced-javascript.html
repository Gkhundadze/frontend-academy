1. Scope and Hoisting

Scope determines the accessibility (visibility) of variables.

Block Scope vs Function Scope
function scopeTest() {
  if (true) {
    var functionScoped = "I am visible throughout the function";
    let blockScoped = "I am only visible inside this if block";
  }
  
  console.log(functionScoped); // Works!
  // console.log(blockScoped); // ReferenceError: blockScoped is not defined
}
scopeTest();

2. Closures

A closure is a function that remembers its outer variables and can access them even after the outer function has finished executing.

Closure Example
function createCounter() {
  let count = 0; // This variable is protected by the closure
  
  return function() {
    count++;
    console.log(count);
  }
}

const counterA = createCounter();
counterA(); // 1
counterA(); // 2

const counterB = createCounter(); // Creates a completely separate scope
counterB(); // 1

3. The 'this' Keyword

In JavaScript, `this` refers to the object that is executing the current function.

Understanding Context
const user = {
  name: 'Alice',
  greet: function() {
    console.log(`Hello, my name is ${this.name}`);
  },
  // Arrow functions do NOT bind their own 'this'
  greetArrow: () => {
    console.log(`Hello, my name is ${this.name}`);
  }
};

user.greet(); // "Hello, my name is Alice"
user.greetArrow(); // "Hello, my name is undefined" (this points to window)
